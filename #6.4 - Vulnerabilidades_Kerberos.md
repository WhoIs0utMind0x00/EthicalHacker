# Vulnerabilidades de Kerberos
Un atacante puede utilizar un ataque de este tipo para manipular los tickets de oro de Kerberos en función de los hashes disponibles. El atacante solo necesita comprometer un sistema vulnerable y obtener las credenciales de usuario local y los hashes de contraseñas. Si el sistema está conectado a un dominio, el atacante puede identificar un hash de contraseña de ticket de concesión de tickets Kerberos (KRBTGT) para obtener el ticket de oro.<br>
Otra debilidad en las implementaciones de Kerberos es el uso de la delegación no restringida de Kerberos, una característica que permite a una aplicación reutilizar las credenciales del usuario final para acceder a recursos alojados en un servidor diferente. Por lo general, solo debe permitir la delegación de Kerberos en un servidor de aplicaciones que, en última instancia, sea de confianza. Sin embargo, esto podría tener consecuencias de seguridad negativas si se abusa, por lo que Active Directory tiene la delegación de Kerberos desactivada de manera predeterminada.<br>
