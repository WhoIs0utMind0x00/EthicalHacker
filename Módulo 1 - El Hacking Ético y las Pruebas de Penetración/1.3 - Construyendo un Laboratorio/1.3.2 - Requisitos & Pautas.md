# 1.3.2 - Requisitos y Pautas para los Laboratorios de Pruebas de Penetración
Ahora profundicemos un poco más en cómo podría ser un entorno de laboratorio de pruebas de penetración y algunas de las mejores prácticas para configurar dicho laboratorio. La siguiente tabla contiene una lista de requisitos y pautas para un entorno típico de pruebas de penetración.

__Red cerrada.__ Debe garantizar el acceso controlado hacia y desde el entorno del laboratorio y el acceso restringido a Internet.<br>
__Entorno informático virtualizado.__ Esto permite una fácil implementación y recuperación de los dispositivos que se están probando.<br>
__Entorno realista.__ Si está organizando un entorno de prueba, debe coincidir con el entorno real lo más posible.<br>
__Monitoreo del estado de salud.__ Cuando algo falla, debe poder determinar por qué sucedió.<br>
__Recursos de hardware suficientes.__ Debe asegurarse de que la falta de recursos no sea la causa de resultados falsos.<br>
__Múltiples sistemas operativos.__ Muchas veces querrá probar o validar un resultado de otro sistema. Siempre es bueno probar desde diferentes sistemas operativos para ver si los resultados difieren.<br>
__Herramientas duplicadas.__ Una excelente manera de validar un hallazgo es ejecutar la misma prueba con una herramienta diferente para ver si los resultados son los mismos.<br>
__Objetivos de práctica.__ Necesitas practicar con tus herramientas. Para ello, debes practicar con objetivos que se sabe que son vulnerables.<br>
