# Ataques de Redireccionamiento
Las direcciones y reenvíos no validados son vulnerabilidades que un atacante puede utilizar para atacar una aplicación web y a sus clientes. El atacante puede explotar tales vulnerabilidades cuando un servidor web acepta una entrada no confiable que podría hacer que la aplicación web redirija la solicitud a una URL contenida en la entrada no confiable. El atacante puede modificar la entrada de URL no confiable y redirigir al usuario a un sitio malicioso para instalar malware o robar información confidencial.<br>

También es posible utilizar vulnerabilidades de redirección y reenvío no validadas para crear una URL que pueda eludir los controles de acceso de la aplicación. Esto, a su vez, permite que un atacante acceda a funciones privilegiadas a las que normalmente no se le permitiría acceder.<br>

__NOTA__: Los ataques de redireccionamiento y reenvío no validados a menudo requieren un poco de ingeniería social.<br>
