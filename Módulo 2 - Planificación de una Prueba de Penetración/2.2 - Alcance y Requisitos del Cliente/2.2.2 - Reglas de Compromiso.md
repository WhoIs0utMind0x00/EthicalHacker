# 2.2.2 - Reglas de Compromiso
El documento de reglas de participación especifica las condiciones en las que se realizará la participación de pruebas de penetración de seguridad. Debe documentar y acordar estas condiciones de la regla de participación con el cliente o una parte interesada adecuada. A continuación, se enumera algunos ejemplos de los elementos que generalmente se incluyen en un documento de reglas de participación.<br>

| Elementos de regla de participación | Ejemplo |
| - | - |
| __Prueba de línea de tiempo__ | Tres semanas, como se especifica en un diagrama de Gantt |
| __Ubicación de la prueba__ | Sede de la empresa en Raleigh, Carolina del Norte |
| __Ventana de tiempo de la prueba (horas del día)__ | de 8:00 AM a 5:00 PM hora del este de los Estados Unidos |
| __Método de comunicación preferido__ | Informe final y reuniones semanales de actualización de estado |
| __Los controles de seguridad que potencialmente podrían detectar o evitar pruebas__ | Sistemas de prevención de intrusiones (IPS), cortafuegos, sistemas de prevención de pérdida de datos (DLP) |
| __Direcciones IP o redes desde las que se originarán las pruebas__ | 10.10.1.0/24, 192.168.66.66, 10.20.15.123 |
| __Tipos de pruebas permitidas o no permitidas__ | Prueba solo de aplicaciones web (app1.secretcorp.org y app2.secretcorp.org).<br>No se permiten ataques de ingeniería social.<br>No se permiten ataques de inyección SQL en el entorno de producción. La inyección SQL solo se permite en los entornos de desarrollo y preparación en:<br>app1-dev.secretcorp.org<br>app1-stage.secretcorp.org<br>app2-dev.secretcorp.org<br>app2-stage.secretcorp.org |

Los diagramas de Gantt y las estructuras de desglose del trabajo (WBS) se pueden usar como herramientas para demostrar y documentar la línea de tiempo. La imagen muestra un ejemplo de diagrama de Gantt básico.<br>
