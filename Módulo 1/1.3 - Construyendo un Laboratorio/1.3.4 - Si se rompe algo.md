# 1.3.4 - ¿Qué pasa si rompes algo?
Poder recuperar el entorno de laboratorio es importante por muchas razones. Como se mencionó anteriormente, al realizar pruebas de penetración, se romperán cosas; a veces, cuando se rompen las cosas, no se recuperan por sí solas. Por ejemplo, cuando está probando aplicaciones web, algunos de los ataques que envía ingresarán datos falsos en campos de formulario, y esos datos probablemente terminarán en la base de datos, por lo que su base de datos se llenará con esos datos falsos. Obviamente, en un entorno de producción, esto no es bueno. Los datos que se ingresan también pueden ser de naturaleza maliciosa, como secuencias de comandos y ataques de inyección. Esto también puede dañar la base de datos. Por supuesto, sabe que esto sería un problema en un entorno de producción. También es un problema en un entorno de laboratorio si no tiene una manera fácil de recuperarse. Sin un método de recuperación rápida, es probable que se quede bloqueado al reconstruir el sistema que se está probando. Esto puede llevar mucho tiempo y, si lo hace para un cliente, puede afectar su línea de tiempo general.

El uso de algún tipo de entorno virtual es ideal, ya que ofrece instantáneas y funciones de restauración para el estado del sistema. Sin embargo, a veces esto no es posible. Por ejemplo, puede estar probando un sistema que no se puede virtualizar. En tal caso, se requiere una copia de respaldo completa del sistema o entorno. De esta manera, puede realizar una copia de respaldo rápidamente y probar si algo se rompe, porque lo más probable es que lo haga. Después de todo, está realizando una prueba de penetración.
