# 7.1.5 - Ataques al Servicio de Metadatos
Tradicionalmente, los desarrolladores de software utilizaban credenciales codificadas de forma fija para acceder a diferentes servicios, como bases de datos y archivos compartidos en un servidor FTP. Para reducir la exposición de tales prácticas inseguras, los proveedores de la nube (como Amazon Web Services [AWS]) han implementado _servicios de metadatos_. Cuando una aplicación requiere acceso a recursos específicos, puede consultar el servicio de metadatos para obtener un conjunto de credenciales de acceso temporal. Este conjunto temporal de credenciales se puede usar para acceder a servicios como los depósitos de AWS Simple Cloud Storage (S3) y otros recursos. Además, estos servicios de metadatos se utilizan para almacenar los datos del usuario suministrados al iniciar una nueva máquina virtual, como Amazon Elastic Compute Cloud o una instancia de AWS EC2 y configurar la aplicación durante la creación de instancias.<br>
Por tanto, los servicios de metadatos son algunos de los servicios más atractivos en AWS para que acceda un atacante. Si puede acceder a estos recursos, como mínimo, obtendrá un conjunto de credenciales de AWS válidas para interactuar con la API. Los desarrolladores de software a menudo incluyen información confidencial en los scripts de inicio de los usuarios. Se puede acceder a estos scripts de inicio de usuario a través de un servicio de metadatos y permiten el lanzamiento de instancias de AWS EC2 (o servicios similares con otros proveedores de la nube) con ciertas configuraciones. A veces, los scripts de inicio incluso contienen nombres de usuario y contraseñas que se utilizan para acceder a diversos servicios.<br>
Mediante el uso de herramientas como [nimbostratus](https://github.com/andresriancho/nimbostratus) puede encontrar vulnerabilidades que podrían conducir a **_ataques a servicios de metadatos_**.<br>
__CONSEJO__: Cuando esté realizando pruebas de penetración de una aplicación web, busque la funcionalidad que obtenga datos de la página y los devuelva al usuario final (de manera similar a como lo haría un proxy). El servicio de metadatos no requiere ningún parámetro en particular. Si accede a la URL *https://x.x.x.x/latest/meta-data/iam/security-credentials/IAM_USER_ROLE_HERE*, devolverá los valores de AccessKeyID, SecretAccessKey y token que necesita para autenticarse en la cuenta.<br>
