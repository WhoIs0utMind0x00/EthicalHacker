# 8.1.1 - Descripción General
### Soluciones de seguridad Protego
Los atacantes no solo realizan explotaciones puntuales. Si tienen un buen acceso y parece que pueden ingresar a las bases de datos y los espacios de archivos que almacenan contenido confidencial, se quedarán. Pueden hacer muchas cosas con el tiempo, si tienen acceso persistente. La exfiltración de datos puede llevar mucho tiempo. Para evitar la detección, los hackers pueden dividir los archivos que están robando en partes muy pequeñas y enviarlos ocultos en las consultas del DNS. A veces, se necesitan meses para robar los archivos.<br>
Hay muchas maneras de mantener una presencia en una red hackeada. Nos gusta establecer un canal de comunicación persistente con al menos un dispositivo explotado para mostrar al cliente que no está detectando dichas explotaciones lo suficientemente rápido como para evitar daños.<br>
Es importante aprender cómo mantener el acceso persistente y los tipos de cosas que conducirán a más explotaciones una vez que se haya obtenido el acceso.<br>
<hr>
Después de la fase de explotación, debe mantenerse firme en un sistema comprometido para realizar tareas adicionales, como instalar o modificar servicios para conectarse de nuevo al sistema comprometido. Puede mantener la persistencia de un sistema comprometido de varias maneras, incluidas las siguientes:<br>

+ Creación de un shell de enlace o inverso
+ Crear y manipular trabajos y tareas programados
+ Creación de daemons y procesos personalizados
+ Creación de nuevos usuarios
+ Creación de puertas traseras adicionales<br>

Cuando mantiene la persistencia en un sistema comprometido, puede realizar varias acciones, como las siguientes:<br>

+ Cargar herramientas adicionales
+ Uso de herramientas del sistema local
+ Realizar escaneos ARP y barridos de ping
+ Realización de la enumeración de servicios de directorio y DNS
+ Lanzar ataques de fuerza bruta
+ Realizar enumeraciones adicionales de usuarios, grupos, bosques, datos confidenciales y archivos sin cifrar
+ Realizar manipulación del sistema mediante protocolos de administración (por ejemplo, WinRM, WMI, SMB, SNMP) y credenciales comprometidas
+ Ejecución de explotaciones adicionales

También puede realizar varias acciones a través del sistema comprometido, incluidas las siguientes:<br>

+ Configuración de reenvío de puertos
+ Creación de túneles SSH o proxies para comunicarse con la red interna
+ Usar una VPN para acceder a la red interna
