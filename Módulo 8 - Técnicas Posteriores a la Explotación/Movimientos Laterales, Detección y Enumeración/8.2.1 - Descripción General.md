# 8.2.1 - Descripción General

### Soluciones de Seguridad Protego
Después de haber aprovechado una vulnerabilidad que nos ha permitido abrir un canal de comunicación persistente con un dispositivo en la red de la organización, es hora de ver qué podemos hacer. Obviamente, no queremos causar ningún daño real, pero queremos confirmar que existen las condiciones en las que podemos controlar las máquinas internas de forma remota, robar o alterar los archivos almacenados.<br>
Una forma de hacerlo es intentar movernos lateralmente de un host a otro en la red. Esto nos permite ver cada dispositivo de forma individual.<br>
Pasar de un host a otro no es suficiente. También necesitamos obtener privilegios en el host para llevar a cabo más explotaciones. Si bien podemos enumerar las contraseñas en los recursos compartidos de Windows con bastante facilidad, existen otras técnicas que a veces debemos emplear para obtener acceso completo a un sistema. Una vez allí, podemos demostrar cómo los actores de amenazas podrían realizar la exfiltración de datos discretos y ampliar su acceso a otros sistemas. Cuando termina nuestra interacción con el cliente, escribimos un informe sobre nuestros hallazgos y, a veces, le mostramos al personal de seguridad dónde hemos estado y qué hemos hecho con videos de demostración.<br>
<hr>

El movimiento lateral (también denominado pivote) es una técnica posterior a la explotación que se puede realizar mediante muchos métodos diferentes. El objetivo principal del movimiento lateral es pasar de un dispositivo a otro para evitar la detección, robar datos confidenciales y mantener el acceso a los dispositivos para exfiltrar los **_datos confidenciales_**, que son datos cuyo robo tendría un impacto severo en la organización. Por lo general, dichos datos no deben compartirse ampliamnete de manera interna o externa. El acceso a los datos confidenciales debe ser limitado y estrictamente controlado. La **_exfiltración de datos_** es el acto de mover deliberadamente datos confidenciales desde el interior de una organización hacia fuera del perímetro sin permiso.<br>
__NOTA__: La transferencia de hash es un ejemplo de una técnica posterior a la explotación que puede usarse para moverse lateralmente y comprometer otros sistemas en la red. Debido a que los hash de contraseña no se pueden revertir, en lugar de intentar averiguar cuál es la contraseña del usuario, un atacante puede usar un hash de contraseña recopilado de un sistema comprometido y luego usar el mismo hash para iniciar sesión en otro sistema cliente o servidor.<br>
