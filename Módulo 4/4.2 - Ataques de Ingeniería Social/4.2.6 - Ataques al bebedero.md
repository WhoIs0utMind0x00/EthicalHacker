# 4.2.6 - Ataques al bebedero
Un ataque al bebedero es un ataque dirigido que se produce cuando un atacante perfila sitios web a los que accede la víctima prevista. Luego, el atacante escanea esos sitios web en busca de posibles vulnerabilidades. Si el atacante localiza un sitio web que puede verse comprometido, el sitio web se inyecta con JavaScript u otro código similar diseñado para redirigir al usuario cuando regresa a ese sitio (Esta redirección también se conoce como _ataque pivote_). Luego, el usuario es redirigido a un sitio con algún tipo de código de explotación. El propósito es infectar las computadoras en la red de la organización, lo que permite que el atacante se establezca en la red para espionaje u otras razones.<br>
Los ataques de bebedero a menudo están diseñados para perfiles de usuarios de organizaciones específicas. Por lo tanto, las organizaciones deberían desarrollar políticas para prevenir estos ataques. Una política de este tipo podría requerir, por ejemplo, la actualización periódica de las aplicaciones antimalware y el uso de navegadores virtuales seguros que tengan poca conectividad con el resto del sistema y la red. Para evitar que un sitio web se vea comprometido como parte de un ataque de este tipo, el administrador debe utilizar métodos de programación adecuados y escanear el sitio web de la organización en busca de malware con regularidad. La educación del usuario es fundamental para ayudar a prevenir este tipo de ataques.<br>
