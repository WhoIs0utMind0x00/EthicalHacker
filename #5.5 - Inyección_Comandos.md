# Vulnerabilidades de Inyección de Comandos
Una **_inyección de comandos_** es un ataque en el que un atacante intenta ejecutar comandos que se supone que no puede ejecutar en un sistema a través de una aplicación vulnerable. Los ataques de inyección de comandos son posibles cuando una aplicación no valida los datos proporcionados por el usuario (por ejemplo, datos ingresados en formularios web, cookies, encabezados HTTP y otros elementos). El sistema vulnerable pasa esos datos a un shell del sistema.<br>
Con la inyección de comandos, un atacante intenta enviar comandos del sistema operativo para que la aplicación pueda ejecutarlos con los privilegios de la aplicación vulnerable.<br>
La inyección de comandos contra aplicaciones web no es tan popular como solía ser porque los marcos de aplicaciones modernos tienen mejores defensas contra estos ataques. La siguiente imagen muestra un ejemplo de inyección de comandos con la aplicación intencionalmente vulnerable DVWA.<br>
