# 6.7.1 - Descripción General

### Soluciones de seguridad Protego
La secuencia de comandos entre sitios (XSS) es un tipo de ataque de inyección en el que las aplicaciones web aceptan secuencias de comandos maliciosas que a menudo se agregan a una URL o se insertan en el texto proporcionado por el usuario que se muestra a todos los visitantes de una página web. Los ataques de inyección se encuentran entre los principales riesgos de seguridad en el Top 10 de OWASP, con XSS como un contribuyente importante. Debemos verificar las vulnerabilidades de XSS cuando nos contratan para probar la seguridad de las aplicaciones web.<br>
Pixel Paradise ejecuta una aplicación web en la tienda para vender juegos y productos coleccionables a los fanáticos de sus productos. También acepta reseñas de sus clientes sobre sus productos. Necesitamos determinar que estas aplicaciones no son vulnerables a XSS.<br>
<hr>

Las vulnerabilidades de **_secuencias de comandos entre sitios (XSS)_**, que se han convertido en una de las vulnerabilidades de aplicaciones web más comunes, se logran mediante los siguientes tipos de ataques:<br>
- XSS reflejado
- XSS almacenado (persistente)
- XSS basado en DOM

La explotación exitosa podría resultar en la instalación o ejecución de código malicioso, compromiso de cuentas, secuestro de cookies de sesión, revelación o modificación de archivos locales, o redirección del sitio.<br>

Normalmente, las vulnerabilidades de XSS se encuentran en lo siguiente:<br>
- Campos de búsqueda que devuelven una cadena de búsqueda al usuario
- Encabezados HTTP
- Campos de entrada que repiten los datos del usuario
- Mensajes de error que devuelven texto proporcionado por el usuario
- Campos ocultos que pueden incluir datos de entrada del usuario
- Aplicaciones (o sitios web) que muestran datos proporcionados por el usuario

El siguiente ejemplo muestra una prueba de XSS que se puede realizar desde la barra de direcciones del navegador:<br>

```
    javascript:alert("Omar_s_XSS test");
    javascript:alert(document.cookie);
```

El siguiente ejemplo muestra una prueba XSS que se puede realizar en un campo de entrada de usuario en un formulario web:<br>

```JavaScript
    <script>alert("XSS Test")</script>
```

__NOTA__: Los atacantes pueden utilizar técnicas de ofuscación en ataques XSS codificando etiquetas o partes maliciosas del script usando Unicode para que el enlace o el contenido HTML quede oculto para el usuario final que navega por el sitio.<br>
