# 6.12.6 - Condiciones de Carrera
Una _condición de carrera_ se produce cuando un sistema o una aplicación intenta realizar dos o más operaciones al mismo tiempo. Sin embargo, debido a la naturaleza de dicho sistema o aplicación, las operaciones deben realizarse en la secuencia adecuada para que se realicen correctamente. Cuando un atacante explota una vulnerabilidad de este tipo, tiene una pequeña ventana de tiempo entre la entrada en vigencia de un control de seguridad y el momento en que se realiza el ataque. La complejidad del ataque en condiciones de carrera es muy alta. En otras palabras, las condiciones de carrera son muy difíciles de explotar.<br>
__NOTA__: Las condiciones de carrera también se denominan **_ataques de tiempo de verificación a tiempo de uso (TOCTOU)_**.<br>
Un ejemplo de condición de carrera es un sistema de administración de seguridad (como un firewall o un sistema de prevención de intrusiones) de modo que el proceso reconstruye las listas y reglas de control de acceso desde el sistema. Un atacante puede tener una ventana de tiempo muy pequeña para eludir esos controles de seguridad hasta que surten efecto en el dispositivo administrado.<br>
