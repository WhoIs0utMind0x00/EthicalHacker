# 6.12.8 - Elementos Ocultos
Los ataques de alteración de los parámetros de las aplicaciones web se pueden ejecutar mediante la manipulación de los parámetros intercambiados entre el cliente web y el servidor web para modificar los datos de la aplicación. Esto podría lograrse manipulando las cookies y abusando de los campos ocultos del formulario.<br>
Podría ser posible alterar los valores almacenados por una aplicación web en campos de formulario ocultos. Veamos un ejemplo de formulario HTML oculto. Supongamos que lo siguiente es parte de un sitio de comercio electrónico que vende productos a clientes en línea:<br>

```HTML
    <input type="hidden" id="123" name="price" value="100.00">
```

En el campo oculto que se muestra en este ejemplo, un atacante podría editar la información de __value__ para reducir el precio de un artículo. No todos los campos ocultos son malos; en algunos casos, son útiles para la aplicación e incluso se pueden usar para proteger contra ataques CSRF.<br>
