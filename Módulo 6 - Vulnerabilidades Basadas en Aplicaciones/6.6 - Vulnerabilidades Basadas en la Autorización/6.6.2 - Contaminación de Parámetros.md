# 6.6.2 - Contaminación de Parámetros
Las vulnerabilidades de contaminación de parámetros HTTP (HPP) pueden introducirse si múltiples parámetros HTTP tienen el mismo nombre. Este problema puede hacer que una aplicación interprete los valores incorrectamente. Un atacante puede explotar las vulnerabilidades de HPP para eludir la validación de entrada, provocar errores en la aplicación o modificar valores de variables internas.<br>
__NOTA__: Las vulnerabilidades de HPP pueden provocar ataques del lado del servidor y del cliente.<br>

Un atacante puede encontrar vulnerabilidades de HPP buscando formularios o acciones que permitan la entrada proporcionada por el usuario. Luego, el atacante puede agregar el mismo parámetro a los datos __GET__ o __POST__, pero con un valor diferente:<br>

```
    https://store.h4cker.org/?search=cars&results=20
```

Esta URL tiene la cadena de consulta __search__ y el valor de parámetro __cars__. El parámetro puede estar oculto entre otros varios parámetros. Un atacante podría dejar el parámetro actual y agregar un duplicado, como se muestra aquí:<br>

```
    https://store.h4cker.org/?search=cars&results=20&search=bikes
```
Después de enviar la solicitud, el atacante podría analizar la página de respuesta para identificar si la aplicación analizó alguno de los valores ingresados. A veces es necesario enviar tres solicitudes HTTP para cada parámetro HTTP. Si la respuesta del tercer parámetro es diferente a la del primero y del segundo, esto puede ser un indicador de una falta de coincidencia de impedancia que podría abusarse para desencadenar vulnerabilidades en HPP.<br>
