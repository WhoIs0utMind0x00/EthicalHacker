# 6.4.4 - Vulnerabilidades de Inyección del Protocolo Ligero de Acceso a Directorios (LDAP)
Las **_vulnerabilidades de inyección LDAP_** son vulnerabilidades de validación de entrada que un atacante utiliza para inyectar y ejecutar consultas en servidores LDAP. Un ataque exitoso de inyección LDAP puede permitir que un atacante obtenga información valiosa para futuros ataques a bases de datos y aplicaciones internas.<br>
De manera similar a la inyección SQL y otros ataques de inyección, los ataques de Inyección LDAP aprovechan las vulnerabilidades que se producen cuando una aplicación inserta una entrada de usuario no sanitizada directamente en una instrucción de LDAP. Al enviar paquetes LDAP modificados, los atacantes pueden hacer que el servidor LDAP ejecute una variedad de consultas y otras declaraciones LDAP. Las vulnerabilidades de inyección LDAP podrían, por ejemplo, permitir que un atacante modifique el árbol de LDAP y modifique información crítica para el negocio.<br>
Hay dos tipos generales de inyección de LDAP:<br>
- __Omisión de autenticación__: Los ataques de inyección LDAP más básicos se inician para omitir la verificación de contraseñas y credenciales.<br>
- __Divulgación de información__: Un atacante podría inyectar paquetes LDAP modificados para enumerar todos los recursos en el directorio de una organización y realizar un reconocimiento.<br>
