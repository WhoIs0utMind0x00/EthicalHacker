# 6.4.1 - Descripción General

### Soluciones de seguridad Protego
Parte de la prueba de una aplicación web es verificar varias vulnerabilidades de inyección. Un buen lugar para buscar es cualquier página web que acepte entradas de usuarios, como credenciales de inicio de sesión, formularios de respuesta de contacto, formularios de publicación en foros, etc. Los formularios de entrada proporcionan una manera de introducir código malicioso en los sistemas. A veces, el código afectará a una sola página web. Otras veces, el código inyectado puede dar acceso a información privilegiada que se almacena en bases de datos. Tenemos la intención de desafiar varios formularios web y casillas de inicio de sesión que hemos descubierto en el sitio de Pixel Paradise.<br>
Con suerte, los codificadores de aplicaciones web de Pixel Paradise tomaron medidas contra las vulnerabilidades de inyección en su aplicación web mediante la creación de filtros que neutralicen estas amenazas. Estas medidas incluyen sanitizar elementos maliciosos del texto de entrada o rechazar por completo cualquier entrada potencialmente peligrosa. Parte de las pruebas de vulnerabilidades de inyección de código es desafiar este tipo de defensa.<br>
Un tipo de ataque realmente peligroso implica inyectar declaraciones SQL para explotar bases de datos. Para probar las vulnerabilidades de la inyección SQL, debe tener un buen conocimiento de SQL. Se centrará en aumentar su familiaridad con SQL y luego investigará otros tipos de ataques en preparación para esta fase de nuestra prueba de penetración.<br>
<hr>
Cambiemos de tema un poco y veamos las vulnerabilidades basadas en inyecciones y cómo explotarlas. Un atacante se aprovecha de las vulnerabilidades de inyección de código inyectando código en un sistema vulnerable para cambiar el curso de la ejecución y forzar a una aplicación o un sistema a procesar datos inválidos. La explotación exitosa puede conducir a la divulgación de información confidencial, manipulación de datos, condiciones de denegación de servicio (DoS) y más. Los siguientes son ejemplos de vulnerabilidades basadas en inyecciones que se analizan en las siguientes secciones:<br>
- Vulnerabilidades de Inyección SQL
- Vulnerabilidades de Inyección HTML
- Vulnerabilidades de Inyección de Comandos
- Vulnerabilidades de Inyección del Protocolo Ligero de Acceso a Directorios (LDAP)<br>
